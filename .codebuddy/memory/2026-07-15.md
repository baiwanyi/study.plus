## server/src/index.ts 定时还款幂等性修复

## 费曼对话表缺失导致查询失败（db:migrate 修复）

现象：`GET /api/feynman/:id/messages` 与 `POST /api/feynman/:id/follow-up` 报错 `Failed query: select ... from "feynman_conversations"`。
根因：`feynman_cards`/`feynman_conversations`/`feynman_messages` 三表已在 `schema.ts` 与 `migrate.ts` 定义，但服务器启动不自动跑迁移（迁移是独立命令 `pnpm db:migrate`），现有 `server/data/study.db` 是在这些表定义前创建的，故缺表。
修复：运行 `pnpm db:migrate`（使用 `CREATE TABLE IF NOT EXISTS`，幂等、不破坏已有数据），三表已成功创建。无需重启服务器（Drizzle 每次直接查库，不缓存表结构）。
经验：以后新增/修改数据库表后必须运行 `pnpm db:migrate` 同步到数据库。

- 修复过程中曾因失败替换残留一处孤立 `} catch`（无 `try` 配对、消息误写 by-grade），已重写两个处理器修正缩进与 catch 消息。

## 费曼对话表缺失导致查询失败（db:migrate 修复）

现象：`GET /api/feynman/:id/messages` 与 `POST /api/feynman/:id/follow-up` 报错 `Failed query: select ... from "feynman_conversations"`。
根因：`feynman_cards`/`feynman_conversations`/`feynman_messages` 三表已在 `schema.ts` 与 `migrate.ts` 定义，但服务器启动不自动跑迁移（迁移是独立命令 `pnpm db:migrate`），现有 `server/data/study.db` 是在这些表定义前创建的，故缺表。
修复：运行 `pnpm db:migrate`（使用 `CREATE TABLE IF NOT EXISTS`，幂等、不破坏已有数据），三表已成功创建。无需重启服务器（Drizzle 每次直接查库，不缓存表结构）。
经验：以后新增/修改数据库表后必须运行 `pnpm db:migrate` 同步到数据库。

- **逻辑缺陷**：`scoreComposition` 中 `Number(result.score) || 75` 在 score 为 `0` 时误判为假值回退为 75，改为 `Number.isFinite(Number(result.score)) ? Number(result.score) : 75`。

根因：`feynman_cards`/`feynman_conversations`/`feynman_messages` 三表已在 `schema.ts` 与 `migrate.ts` 定义，但服务器启动不自动跑迁移（迁移是独立命令 `pnpm db:migrate`），现有 `server/data/study.db` 是在这些表定义前创建的，故缺表。
修复：运行 `pnpm db:migrate`（使用 `CREATE TABLE IF NOT EXISTS`，幂等、不破坏已有数据），三表已成功创建。无需重启服务器（Drizzle 每次直接查库，不缓存表结构）。
经验：以后新增/修改数据库表后必须运行 `pnpm db:migrate` 同步到数据库。
