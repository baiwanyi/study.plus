## 1. 文档结构规范

- 每个文档**必须**有且仅有一个一级标题（`# 标题`），作为文档主标题。
- 标题层级**不应跳跃**（例如：三级标题后直接使用五级标题）。
- 建议文档长度超过 500 行时，在开头添加**目录**（通过 `[TOC]` 或手动链接）。
- 使用水平分割线 `---` 分隔大的章节（前后需有空行）。

**示例：**

```markdown
# 文档主标题

## 二级标题

### 三级标题

---

## 另一个二级标题
```

**错误示例：**

```markdown
# 文档主标题

### 跳过了二级标题 ← 不推荐

##### 跳过了四级标题 ← 不推荐
```

## 2. 标题语法与规范

- 使用 `#` 表示标题，`#` 的数量代表标题层级（1~6 级）。
- `#` 与标题文字之间**必须有一个空格**。
- 标题前后建议各留一个空行，保证渲染一致性。
- 不推荐使用下划线风格的标题（`===` 和 `---`），统一使用 `#` 风格。

**示例：**

```markdown
# 一级标题

## 二级标题

### 三级标题

#### 四级标题

##### 五级标题

###### 六级标题
```

**错误示例：**

```markdown
#没有空格 ← 错误，# 后缺少空格

一级标题
===  ← 不推荐，应使用 # 风格
```

## 3. 段落与换行

- 段落之间使用**一个空行**分隔。
- 行内换行：在行末添加**两个空格**再回车，或使用 `<br>` 标签。
- 不要连续使用多个空行，保持文档紧凑。
- 段落开头**不要**使用空格或 Tab 缩进（缩进会被解析为代码块）。

**示例：**

```markdown
这是第一段文字。

这是第二段文字。段落之间用空行分隔。

这是第三段的第一行（行末有两个空格）
这是第三段的第二行（软换行）

这是第四段，使用 <br> 标签换行<br>这是同一段的下一行
```

**渲染效果：**

这是第一段文字。

这是第二段文字。段落之间用空行分隔。

这是第三段的第一行（行末有两个空格）
这是第三段的第二行（软换行）

**错误示例：**

```markdown
    缩进四个空格会被解析为代码块 ← 错误
```

## 4. 文本强调

| 语法 | 效果 | 用途 |
|------|------|------|
| `*斜体*` 或 `_斜体_` | *斜体* | 术语、外文等 |
| `**粗体**` 或 `__粗体__` | **粗体** | 重点强调 |
| `***粗斜体***` 或 `___粗斜体___` | ***粗斜体*** | 极强强调 |
| `~~删除线~~` | ~~删除线~~ | 标记废弃内容 |
| `~下标~` | H~2~O | 下标（需渲染器支持） |
| `^上标^` | X^2^ | 上标（需渲染器支持） |

- 推荐统一使用 `*` 风格（`*斜体*`、`**粗体*`*、`***粗斜体***`），避免与 `_` 在链接中的用法混淆。
- 强调标记内部不要有多余空格。

**示例：**

```markdown
这是*斜体*文字，这是**粗体**文字，这是***粗斜体***文字。

这是~~已废弃的内容~~，已经不再使用。

化学式：H~2~O，数学公式：X^2^
```

**渲染效果：**

这是*斜体*文字，这是**粗体**文字，这是***粗斜体***文字。

这是~~已废弃的内容~~，已经不再使用。

## 5. 列表

### 5.1 无序列表

- 使用 `-`、`*` 或 `+` 作为标记，项目内保持**统一符号**。
- 推荐统一使用 `-`。

**示例：**

```markdown
- 项目一
- 项目二
- 项目三
```

**渲染效果：**

- 项目一
- 项目二
- 项目三

### 5.2 有序列表

- 使用数字加 `.` 表示，数字顺序不影响渲染结果（渲染器自动编号），但建议按顺序书写。
- 列表项之间不需要空行。

**示例：**

```markdown
1. 第一步
2. 第二步
3. 第三步
```

**渲染效果：**

1. 第一步
2. 第二步
3. 第三步

### 5.3 嵌套列表

- 子列表**缩进 2~4 个空格**（推荐 2 个空格），与父列表项对齐。

**示例：**

```markdown
- 水果
  - 苹果
  - 香蕉
- 蔬菜
  - 白菜
  - 萝卜
```

**渲染效果：**

- 水果
  - 苹果
  - 香蕉
- 蔬菜
  - 白菜
  - 萝卜

### 5.4 列表中的段落与代码块

- 列表中的段落需要缩进与列表项对齐。
- 列表中的代码块需要缩进 2 个空格（无序列表）或 3 个空格（有序列表）。

**示例：**

```markdown
1. 第一项的说明

   第一项的补充段落（缩进 3 个空格）

       代码块内容（缩进 5 个空格 + 4 个空格代码缩进）

2. 第二项
```

## 6. 链接与图片

### 6.1 行内链接

**语法：** `[链接文字](URL "可选标题")`

```markdown
[访问 GitHub](https://github.com "GitHub 官网")

[百度搜索](https://www.baidu.com)
```

**渲染效果：**

[访问 GitHub](https://github.com "GitHub 官网")

### 6.2 引用式链接

适用于同一链接多次引用的场景，保持正文简洁。

```markdown
[GitHub][1] 和 [GitHub 官网][1] 是同一个链接。

[1]: https://github.com "GitHub 官网"
```

### 6.3 自动链接

URL 和邮箱可自动转换为可点击链接：

```markdown
<https://www.example.com>

<contact@example.com>
```

**渲染效果：**

<https://www.example.com>

### 6.4 图片

**语法：** `![替代文字](图片URL "可选标题")`

```markdown
![Logo 图片](https://via.placeholder.com/150 "示例 Logo")
```

**渲染效果：**

![Logo 图片](https://via.placeholder.com/150 "示例 Logo")

### 6.5 带链接的图片

```markdown
[![Logo](https://via.placeholder.com/100 "点击访问")](https://github.com)
```

### 6.6 图片尺寸控制（HTML 扩展）

标准 Markdown 不支持调整图片大小，可使用 HTML：

```markdown
<img src="https://via.placeholder.com/150" width="100" alt="缩放图片" />
```

## 7. 代码块与行内代码

### 7.1 行内代码

- 使用反引号 `` ` `` 包裹代码片段。
- 如果代码中包含反引号，使用双反引号 `` `` `` 包裹。

**示例：**

```markdown
使用 `console.log()` 输出日志。

代码中包含反引号：`` `反引号` ``
```

**渲染效果：**

使用 `console.log()` 输出日志。

### 7.2 围栏式代码块

- 使用三个反引号 ` ``` ` 或三个波浪号 `~~~` 包裹多行代码。
- 在开头的 ` ``` ` 后指定语言标识，启用语法高亮。

**示例：**

````
```javascript
function greet(name) {
  console.log(`Hello, ${name}!`);
}

greet('World');
```
````

**渲染效果：**

```javascript
function greet(name) {
  console.log(`Hello, ${name}!`);
}

greet('World');
```

### 7.3 常用语言标识

| 语言 | 标识 | 语言 | 标识 |
|------|------|------|------|
| JavaScript | `javascript` / `js` | Python | `python` / `py` |
| TypeScript | `typescript` / `ts` | Java | `java` |
| HTML | `html` | CSS | `css` |
| JSON | `json` | SQL | `sql` |
| Shell | `shell` / `bash` | YAML | `yaml` |
| Markdown | `markdown` / `md` | Diff | `diff` |

### 7.4 缩进式代码块（不推荐）

缩进 4 个空格或 1 个 Tab 也会生成代码块，但无法指定语言，不推荐使用：

```markdown
    缩进四格生成代码块
    无法指定语言
```

## 8. 引用

### 8.1 基本用法

- 使用 `>` 表示引用，`>` 与内容之间留一个空格。
- 引用可以嵌套。

**示例：**

```markdown
> 这是一段引用文字。
>
> 引用中的第二段。

> 嵌套引用：
>
> > 这是嵌套的引用内容。
```

**渲染效果：**

> 这是一段引用文字。
>
> 引用中的第二段。

> 嵌套引用：
>
> > 这是嵌套的引用内容。

### 8.2 引用中的其他元素

引用中可以包含列表、代码块等其他 Markdown 元素：

**示例：**

```markdown
> 引用中的列表：
>
> - 项目一
> - 项目二
>
> 引用中的代码：
>
> `const x = 1;`
```

**渲染效果：**

> 引用中的列表：
>
> - 项目一
> - 项目二
>
> 引用中的代码：
>
> `const x = 1;`

## 9. 表格

### 9.1 基本语法

- 使用 `|` 分隔列，使用 `-` 分隔表头与内容。
- `:` 控制对齐方式：`---` 左对齐、`:---:` 居中、`---:` 右对齐。
- 表格前后建议各留一个空行。

**示例：**

```markdown
| 姓名 | 年龄 | 城市 |
|------|------|------|
| 张三 | 25 | 北京 |
| 李四 | 30 | 上海 |
| 王五 | 28 | 广州 |
```

**渲染效果：**

| 姓名 | 年龄 | 城市 |
|------|------|------|
| 张三 | 25 | 北京 |
| 李四 | 30 | 上海 |
| 王五 | 28 | 广州 |

### 9.2 对齐方式

```markdown
| 左对齐 | 居中对齐 | 右对齐 |
| :--- | :---: | ---: |
| 内容 | 内容 | 内容 |
| Left | Center | Right |
```

**渲染效果：**

| 左对齐 | 居中对齐 | 右对齐 |
| :--- | :---: | ---: |
| 内容 | 内容 | 内容 |
| Left | Center | Right |

### 9.3 表格中的特殊内容

- 表格中可使用行内代码、强调等行内语法。
- 如需在表格中使用 `|`，使用 `\|` 转义。

```markdown
| 语法 | 说明 |
|------|------|
| `code` | 行内代码 |
| **粗体** | 强调文字 |
| a \| b | 管道符转义 |
```

## 10. 任务列表

- 使用 `- [ ]` 表示未完成，`- [x]` 表示已完成。
- `x` 必须小写。
- 任务列表可与普通列表混用。

**示例：**

```markdown
- [x] 完成需求分析
- [x] 编写技术方案
- [ ] 开发功能模块
- [ ] 编写单元测试
- [ ] 部署上线
```

**渲染效果：**

- [x] 完成需求分析
- [x] 编写技术方案
- [ ] 开发功能模块
- [ ] 编写单元测试
- [ ] 部署上线

  - [x] 配置 CI/CD

  - [ ] 用户模块

  - [x] 配置 CI/CD

## 11. 转义字符

- 使用反斜杠 `\` 转义 Markdown 中的特殊字符，使其显示为普通字符。

**可转义字符：**

```markdown
\   反斜杠
`   反引号
*   星号
_   下划线
{}  花括号
[]  方括号
()  小括号
#   井号
+   加号
-   减号（连字符）
.   句点
!   感叹号
|   管道符
```

**示例：**

```markdown
\*这不是斜体\*

\# 这不是标题

\[这不是链接\]

\\ 反斜杠本身
```

**渲染效果：**

\*这不是斜体\*

\# 这不是标题

\[这不是链接\]

\\ 反斜杠本身

## 12. 注释与隐藏内容

### 12.1 HTML 注释

标准 Markdown 不支持注释语法，使用 HTML 注释实现：

```markdown
<!-- 这是一条注释，不会在渲染结果中显示 -->

<!--
  多行注释
  同样不会显示
-->
```

### 12.2 隐藏内容技巧

通过 HTML 注释可以隐藏任意内容（包括代码片段、待办事项等）：

```markdown
以下内容已发布：

这是可见的正文内容。

<!-- 以下内容暂未发布，作为草稿保留 -->
<!-- - 草稿项一 -->
<!-- - 草稿项二 -->
```

## 13. 最佳实践建议

### 13.1 格式规范

- **统一风格**：同一文档内，列表符号、强调方式等保持一致。
- **空行使用**：标题前后、列表前后、代码块前后、表格前后均留空行。
- **行长度**：建议每行不超过 120 个字符，提升可读性和版本控制友好度。
- **文件编码**：统一使用 UTF-8 编码。
- **换行符**：统一使用 LF（`\n`），避免 CRLF（`\r\n`）。

### 13.2 内容组织

- 一个 Markdown 文件只聚焦**一个主题**。
- 文档开头简要说明目的、适用范围和版本。
- 较长文档添加目录，方便导航。
- 合理使用标题层级，**不超过四级**标题（多数场景下三级即可）。

### 13.3 协作规范

- 使用 `.editorconfig` 统一编辑器配置。
- 使用 Prettier 等 Markdown 格式化工具保持风格统一。
- 避免使用特定平台专属语法（如 GitHub 的 `mermaid`、`math`），除非明确目标平台。
- 图片优先使用相对路径，便于离线查看。

### 13.4 性能与可维护性

- 图片应使用合适尺寸，避免引用超大图片文件。
- 长文档考虑拆分为多个子文档，通过链接互相引用。
- 定期清理失效链接和过期内容。
- 使用有意义的文件名，避免 `Untitled.md`、`新建文档.md` 等。

### 13.5 常见错误对照表

| 错误写法 | 正确写法 | 说明 |
|---------|---------|------|
| `#标题` | `# 标题` | # 后需有空格 |
| `-项目` | `- 项目` | 列表标记后需有空格 |
| `** 粗体 **` | `**粗体**` | 强调标记内不要有空格 |
| `[链接] (url)` | `[链接](url)` | 链接文字与括号间不要有空格 |
| `![图片] (url)` | `![图片](url)` | 图片语法同链接 |
| 连续多个空行 | 最多一个空行 | 多空行无额外效果 |
| Tab 缩进段落 | 不缩进 | 缩进会被解析为代码块 |
